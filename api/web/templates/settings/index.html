Outbound webhook

Mutual posts a small JSON envelope to this URL on lifecycle events: claim.submitted, claim.approved, claim.rejected, claim.paid, and monthly_close.due. Fire-and-log — there are no retries in v0. Failures are recorded in the audit log as webhook.dispatched rows.

Manual trigger

"Monthly close" doesn't fire automatically in v0. Run this on a calendar day that suits your pool — e.g. wire a cron line: 0 9 1 * * curl -fsS -X POST -b cookies.txt https://your-host/pools/{{ pool.slug }}/webhooks/monthly-close
