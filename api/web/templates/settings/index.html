Pool identity

The slug is what shows up in URLs (e.g. /pools/{{ pool.slug }}/...). Renaming the slug updates every link, but ledger entries and audit history are keyed by pool id internally — none of that is lost.

Outbound webhook

Mutual posts a small JSON envelope to this URL on lifecycle events: claim.submitted, claim.approved, claim.rejected, claim.paid, and monthly_close.due. Fire-and-log — there are no retries. Failures are recorded in the audit log as webhook.dispatched rows.

Manual monthly-close trigger

"Monthly close" doesn't fire automatically. Run it on the calendar day that suits your pool — wire e.g. 0 9 1 * * curl -fsS -X POST -b cookies.txt https://your-host/pools/{{ pool.slug }}/webhooks/monthly-close
